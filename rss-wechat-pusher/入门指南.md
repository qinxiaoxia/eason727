# RSS 推送到企业微信 · 入门指南

你已经有：WeWe RSS 部署在 Zeabur，公众号已配置好。  
接下来按下面步骤，把文章自动推到企业微信群。

---

## 第一步：拿到 WeWe RSS 的订阅地址

1. 打开你的 WeWe RSS 网站（Zeabur 给你的地址，比如 `https://wewe-rss.zeabur.app`）
2. 登录后台（需要 AUTH_CODE）
3. 在「公众号源」或「订阅」页面，找到 **订阅地址**
4. 复制 **全部公众号** 的地址，一般是：
   - `https://你的域名/feeds/all.atom`
   - 或 `https://你的域名/feeds/all.rss`

   例如：`https://wewe-rss.zeabur.app/feeds/all.atom`

> 如果是单个公众号，格式类似：`https://你的域名/feeds/MP_WXS_xxxxx.atom`

---

## 第二步：创建企业微信群机器人

1. 打开 **企业微信**，进入要接收推送的群
2. 点击右上角 **···** → **群设置**
3. 找到 **群机器人** → **添加机器人**
4. 给机器人起个名字（如「RSS 推送」）
5. 创建后，**复制 Webhook 地址**，形如：
   ```
   https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```

---

## 第三步：配置 rss-wechat-pusher

1. 打开终端（或 VS Code 终端），进入项目目录：
   ```bash
   cd /Users/hehe/Desktop/备份勿动/vue-admin/rss-wechat-pusher
   ```

2. 复制配置模板：
   ```bash
   cp config.example.py config.py
   ```

3. 用任意编辑器打开 `config.py`，修改这两处：

   **① 企业微信 Webhook**（把 `xxx` 换成你第二步复制的地址）：
   ```python
   WECHAT_WEBHOOK = "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的key"
   ```

   **② RSS 源**：`config.example.py` 已内置 WeWe RSS + 多个网安 RSS，如需修改见示例。

   **③ 大模型分类（可选）**：规则未命中的文章会调用 LLM 细分为「漏洞信息」「重大安全事件」「网安新闻资讯」「网安赛事资讯」「其他资讯」。不配置则全部归为「其他资讯」。
   ```python
   LLM_API_KEY = "sk-xxx"           # DeepSeek / 通义 / 智谱 等 API Key
   LLM_BASE_URL = "https://api.deepseek.com/v1"   # DeepSeek 示例
   LLM_MODEL = "deepseek-chat"
   ```
   支持任意 OpenAI 兼容 API（通义：`https://dashscope.aliyuncs.com/compatible-mode/v1`，智谱等同理）。

4. 保存文件。

---

## 第四步：安装依赖并测试

1. 安装 Python 依赖：
   ```bash
   pip install -r requirements.txt
   ```
   如果 `pip` 报错，试试：`pip3 install -r requirements.txt`

2. 运行脚本：
   ```bash
   python main.py --push-now
   ```
   或：`python3 main.py --push-now`

   > 加 `--push-now` 仅推「其余四类」；`--push-all-now` 一次推实时两类 + 其余四类（全量）。实时两类默认仅在轮巡时段自动推送。

3. 如果配置正确，企业微信群里会收到一条推送消息。

## 第五步：部署

### 推送逻辑（已实现）

| 类型 | 触发条件 | 推送内容 |
|------|----------|----------|
| **轮巡实时** | 新文章被分类为上述两类，且本次运行落在 6:00–20:00 每两小时轮巡窗口内 | 「监管机构预警」「重大安全事件」当天文章，每类一条 |
| **定时推送** | 每天 9:30、15:30（北京） | 仅推送其余四类（9:30 为昨 15:30～今 9:30；15:30 为当天），按类别合并 |
| **去重** | 推送后写入 `pushed` 表 | 避免重复推送 |

### 方式 A：本机 crontab（适合电脑常开）

1. 打开终端，输入 `crontab -e`
2. 在末尾加一行（把路径换成你的实际路径）：
   ```
   */30 * * * * cd /path/to/vue-admin/rss-wechat-pusher && python3 main.py
   ```
3. 保存退出。建议每 15～30 分钟运行一次，以覆盖轮巡整点与 9:30、15:30 定时窗口。

### 方式 B：GitHub Actions（适合不想开电脑）

1. 把 `vue-admin` 项目推送到 GitHub
2. 打开仓库 → **Settings** → **Secrets and variables** → **Actions**
3. 点 **New repository secret**，添加：

   | 名称 | 必填 | 值 |
   |------|------|-----|
   | `WECHAT_WEBHOOK` | ✓ | 企业微信 Webhook 完整地址 |
   | `FEEDS_JSON` | ✓ | `[["https://xxx/feeds/all.atom","wewe_rss"],["https://www.freebuf.com/feed","rss"],...]` |
   | `WEWE_RSS_URL` | | WeWe RSS 地址 |
   | `LLM_API_KEY` | | 大模型 API Key（分类+翻译） |
   | `LLM_BASE_URL` | | 如 `https://dashscope.aliyuncs.com/compatible-mode/v1` |
   | `LLM_MODEL` | | 如 `qwen-turbo` |

4. 保存后，`.github/workflows/rss-push.yml` 按 `cron` 轮巡（北京 6、8、12、14、18 整点，不含 10:00/16:00）+ 定时（9:30、15:30 各一次），UTC 见 workflow 注释。

## 常见问题

**Q：运行报错「请复制 config.example.py 为 config.py」**  
A：说明还没做第三步，先执行 `cp config.example.py config.py` 并编辑。

**Q：企业微信没收到消息**  
A：检查 Webhook 地址是否完整、是否复制错；检查企业微信机器人是否被移除。

**Q：推送的是空内容**  
A：可能是当天没有新文章；或 WeWe RSS 的订阅地址不对，在浏览器打开该地址看是否有内容。

**Q：监管机构预警没有触发**  
A：需作者/公众号名为「国家网络安全通报中心」或「国家互联网应急中心CNCERT」，且标题符合规则。确认 feed 中是否包含作者信息。

**Q：如何启用大模型分类？**  
A：在 `config.py` 中填写 `LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL`。支持 DeepSeek、通义千问、智谱等 OpenAI 兼容 API。留空则全部归为「其他资讯」。

**Q：运行显示「完成」但没有推送**  
A：普通 RSS 文章只在配置的定时档窗口内推送。手动运行加 `--push-now` 可立即推送：
```bash
python main.py --push-now
```

**Q：「重点防范」类文章会提取 IOC 吗？**  
A：公众号为「国家网络安全通报中心」且标题以「重点防范」开头的文章，会抓取正文提取域名与 IP。若有有效 IOC（排除 0.0.0.0、127.x 等），按特殊格式推送（含【相关IOC】）；若无则按基础格式推送。需安装 `beautifulsoup4`。

**Q：企业微信报错「content exceed max length 4096」**  
A：程序已按字节自动分片（单条 3800 字节），超长会拆成多条发送。可设置环境变量 `WECHAT_MAX_BYTES=3500` 进一步缩小单条。
